## What I learned
- A route maps a URL to a controller action, and the action returns a response to the browser.
- `Response` returns raw output directly, while `$this->render()` renders a Twig template and passes data into it.
- Controllers should prepare logic and data, while templates should handle presentation and output.
- Twig variables come from the keys passed into `$this->render()`.
- `extends` means inheritance in PHP classes, and in Twig it means a child template uses a parent layout.
- Symfony Flex adds aliases and recipes on top of Composer, which is why packages can add config and files automatically.
- Symfony’s debug pack adds the web debug toolbar, profiler, logging, and useful `bin/console` debug commands.
- `$this->json()` is the JSON shortcut for controllers, and Symfony’s serializer is needed when returning real objects cleanly as JSON.
