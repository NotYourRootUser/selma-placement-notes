## What I learned
- Symfony supports both API responses with `json()` and HTML rendering with `render()`.
- Controllers should stay thin and delegate data fetching to repositories.
- Route parameters map directly to controller arguments, and constraints like `\d+` enforce valid input.
- `$repository->find($id)` can return an object or `null`, so missing data needs handling.
- Twig templates receive controller data and render HTML using inheritance and blocks.
- Route names and `path()` make URL generation dynamic and avoid hardcoded links.
- A customer-facing dashboard widget should answer a practical user question, not just display technical metrics.
- For a communication widget, “needs attention today” could mean failed messages, unread replies, pending messages, missing phone numbers, invalid numbers, or a low delivery rate.
- The strongest V1 should stay small and use demo data first instead of assuming access to Selma’s real production data.
